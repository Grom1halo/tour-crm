# Инструкция по разделу «Бухгалтерия»

Раздел «Бухгалтерия» состоит из шести вкладок:
**Движение средств · Оплата счетов · Агенты · Сотрудники · Бух. отчёт · Выручка**

---

## 1. Движение средств

### Что это такое

Хронологический журнал всех денежных операций: поступлений от туристов, выплат операторам, расходов на агентов, ручных корректировок.

### Фильтры

| Фильтр | Что делает |
|--------|-----------|
| **Валюта** (THB / VND / Все) | Показывает только записи в выбранной валюте |
| **Категория группы** | «Все движения» — весь журнал; «Только касса» — реальные деньги (без комиссий/списаний); «Операторские и списания» — только внутренние проводки |
| **Метод оплаты** | Фильтр по способу оплаты (наличные, перевод, карта и т.д.) |
| **Дата от / до** | Период отображения. По умолчанию — текущий месяц |

> **Важно:** строка «Итого» под таблицей считается по отфильтрованным записям — она покажет баланс именно по выбранному фильтру.

### Колонки таблицы

| Колонка | Описание |
|---------|---------|
| Дата | Дата операции |
| Категория | Тип операции (Оплата тура, Расход, Комиссия оператору, Зарплата и т.д.) |
| Описание / компания | Вводится вручную или берётся из ваучера |
| Метод | Способ оплаты |
| Приход | Сумма прихода в кассу |
| Расход | Сумма расхода из кассы |
| Баланс | Нарастающий итог (считается по всем записям в выбранном фильтре) |

### Автоматические записи

Следующие операции создаются системой автоматически и вручную их удалять **не рекомендуется**:

- **Оплата тура** — создаётся при добавлении платежа в ваучер
- **Комиссия оператору** — при закрытии периода оператора
- **Списание долга** — при закрытии исторического долга оператора
- **Выплата агенту** — при отметке выплаты агенту
- **Зарплата** — при начислении зарплаты сотруднику

### Добавление записи вручную

Нажмите **«+ Запись»** (правый верхний угол). Заполните:
- Дата, Тип (Приход/Расход), Сумма, Валюта
- Метод оплаты
- Категория
- Компания (необязательно — для операций с оператором)
- Описание / комментарий

### Редактирование и удаление

Каждая строка имеет иконки ✏️ и 🗑️. Автоматические записи лучше не удалять — вместо этого создайте корректирующую запись.

## 2. Оплата счетов (операторы)

### Что это такое

Сводная таблица задолженности/баланса перед каждой компанией-оператором. Здесь видно, кому сколько должны или кто сколько должен нам.

### Колонки таблицы

| Колонка | Описание |
|---------|---------|
| Компания | Название оператора |
| Сумма к оплате | Суммарный долг (все ваучеры, нетто-цены, за вычетом уже оплаченного) |
| Оплачено | Сколько уже перечислено оператору (все выплаты за все время) |
| Баланс | Итог: отрицательный = мы должны оператору, положительный = оператор должен нам или у нас депозит |

**Зелёный цвет** — компания в плюсе (депозит у оператора или переплата).  
**Красный цвет** — есть задолженность перед оператором.

### Переключатель «За всё время / За период»

- **За всё время** — полный исторический баланс с начала работы
- **За период** — только ваучеры в выбранном диапазоне дат (вверху те же фильтры дат)

### Кнопки действий (у каждой компании)

| Кнопка | Действие |
|--------|---------|
| **+ Оплатить** | Создать выплату оператору. Заполните дату, сумму, метод оплаты, комментарий. Сумма уйдёт в «Движение средств» как расход |
| **Списать** | Закрыть исторический период долга. Используется один раз для фиксации начального баланса (например, долг на 01.04.2026). После этого период помечается закрытым |
| **±** | Ручная коррекция: добавить или убрать произвольную сумму из баланса с комментарием (например, чаевые, скидка, возврат). Попадает в «Движение средств» как «Ручная коррекция» |

### Развернуть ваучеры

Нажмите на строку компании — раскроется список всех ваучеров этой компании с суммами и статусами оплаты.

### Закрытие исторического долга

1. Нажмите **«Списать»** у компании
2. Укажите дату отсечения (например, 01.04.2026)
3. Введите сумму долга на эту дату (или оставьте автоматически рассчитанную)
4. Подтвердите

Система создаст запись «Списание долга» и пометит все ваучеры до указанной даты как оплаченные оператором.

> **Важно:** закрытие периода учитывает только ваучеры с датой тура **до** указанной даты. Ваучеры после — остаются в открытом балансе.

## 3. Агенты

### Что это такое

Аналог «Оплаты счетов», но для агентов. Показывает начисленные комиссии и выплаченные суммы.

### Как считается комиссия

Комиссия агента = (цена продажи клиенту − нетто-цена оператора) × доля агента (%)

Доля задаётся в карточке агента.

### Колонки

| Колонка | Описание |
|---------|---------|
| Агент | Имя агента |
| Начислено | Суммарная комиссия по всем ваучерам агента |
| Выплачено | Уже перечислено агенту |
| Баланс | Долг перед агентом (отрицательный) или переплата |

### Кнопки действий

| Кнопка | Действие |
|--------|---------|
| **+ Выплатить** | Зафиксировать выплату комиссии агенту |
| **±** | Ручная коррекция баланса |

### Выплата через ваучер

В каждом ваучере есть флаг «Комиссия агенту выплачена» — его можно поставить вручную, чтобы отметить конкретный ваучер как оплаченный агенту.

## 4. Сотрудники

### Что это такое

Раздел для начисления и учёта зарплат сотрудников компании.

### Список сотрудников

Каждая карточка содержит:
- Имя сотрудника
- Оклад (ежемесячный)
- Историю выплат

### Начисление зарплаты

1. Нажмите **«+ Выплата»** на карточке сотрудника
2. Укажите период (месяц), сумму, дату выплаты, метод оплаты
3. Подтвердите — запись появится в «Движение средств» как категория «Зарплата»

## 5. Бух. отчёт

### Что это такое

Подробная таблица по всем ваучерам с финансовыми показателями. Предназначена для бухгалтерии и сверки.

### Колонки

| Колонка | Описание |
|---------|---------|
| Ваучер | Номер ваучера (кликабельный — открывает карточку) |
| Дата тура | Дата выезда |
| Компания | Оператор |
| Тур | Название тура |
| Продажная цена | Цена клиенту (total_sale) |
| Нетто | Цена оператора (total_net) |
| Прибыль | Продажная − нетто |
| Оплачено | Сколько получено от клиента |
| Остаток | Сколько ещё должен клиент |
| ✓ | Чекбокс подтверждения бухгалтером |

### Фильтры

Доступны те же фильтры дат и валюты, что и в «Движении средств».

### Подтверждение

Галочка «✓» позволяет бухгалтеру отметить строку как проверенную. Это только визуальная метка, на расчёты не влияет.

## 6. Выручка

### Что это такое

Сводный отчёт о прибыльности за период. Считает выручку, расходы и чистую прибыль.

### Структура

**Строка «Выручка»** — суммарная прибыль по всем ваучерам (продажная − нетто) за период.

**Блок расходов** — перечень категорий расходов из «Движения средств». Каждую строку можно включить/выключить галочкой, чтобы посмотреть влияние на итог.

**Ручные расходы** — можно добавить произвольный расход прямо в отчёт (например, аренда офиса), не создавая запись в журнале.

**Итог** = Выручка − Все включённые расходы

## Типовые сценарии

### Турист оплатил тур

1. Откройте ваучер → раздел «Платежи»
2. Нажмите «+ Добавить платёж»
3. Укажите сумму, метод, дату
4. Сохраните → запись автоматически появится в «Движении средств»

### Перечислили деньги оператору

1. Бухгалтерия → «Оплата счетов»
2. Найдите компанию → нажмите «+ Оплатить»
3. Укажите сумму, дату, метод (например, «Банковский перевод»)
4. Сохраните → расход появится в «Движении средств»

### Нужно исправить сумму выплаты оператору

Если сумма указана неверно:
- Вариант 1: удалите запись в «Движении средств» и создайте новую
- Вариант 2: создайте корректирующую запись «±» с нужной дельтой и комментарием

### Депозит в компанию

Если клиент заплатил часть суммы прямо оператору (cash on tour):
- Это поле «Кэш на туре» в ваучере
- Система автоматически учитывает его в балансе ваучера и в расчёте долга перед оператором
- В «Движение средств» можно нажать «Зачислить кэш с тура» — создаст приход в кассу

### Закрытие месяца

1. Убедитесь, что все платежи по ваучерам текущего месяца внесены
2. «Оплата счетов» → для каждой компании нажмите «Списать» (если закрываете исторический долг) или проведите обычные выплаты
3. «Агенты» → выплатите накопившиеся комиссии
4. «Сотрудники» → начислите зарплаты
5. «Выручка» → сверьте итог за месяц

## Частые вопросы (FAQ)

**Q: Почему баланс оператора не 0 после выплаты?**  
A: Возможно, есть ваучеры с нетто-ценой, которые ещё не помечены как оплаченные. Разверните строку компании и проверьте список ваучеров.

**Q: Почему в «Движении средств» показывает баланс 0 или неправильный?**  
A: Проверьте фильтры — возможно, выбран фильтр «Только касса» или «Операторские», который исключает часть записей. Переключите на «Все движения».

**Q: Турист заплатил в другой валюте — где это видно?**  
A: В «Движении средств» переключите фильтр валюты на VND. Балансы по THB и VND считаются отдельно.

**Q: Как проверить, что оператор учёл наш депозит?**  
A: В «Оплата счетов» → строка оператора → баланс должен быть положительным (зелёным) на сумму депозита.

**Q: Куда попадает комиссия агенту?**  
A: В раздел «Агенты» (начисляется автоматически при создании ваучера с агентом). В «Движение средств» попадает только при фактической выплате — кнопка «+ Выплатить».

**Q: Видно только с 14-го числа, хотя платили с 12-го?**  
A: Проверьте фильтр «Дата от» — вероятно, он установлен на 14-е. Измените на нужную начальную дату.

**Q: Депозит ушёл не в ту компанию?**  
A: Платёж типа «Депозит в компанию» привязывается к компании из карточки ваучера. Если оператор в ваучере указан неверно — нужно исправить ваучер, тогда и депозит перевяжется корректно.
